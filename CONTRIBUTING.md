# Contributing

## Filing issues

Include:

- What happened and what you expected.
- Steps to reproduce.
- Version of the tool or library.
- OS and platform.

## Branches and pull requests

- Branch off `main`.
- Use conventional commit prefixes: `feat`, `fix`, `docs`, `chore`, `refactor`, `test`.
- One logical change per PR. Squash merge into `main`.
- PRs that change behavior must include tests.

## Documentation style

When editing docs, follow these rules:

- No emoji.
- No em dashes. Use commas, periods, or parentheses.
- No filler phrases ("dive in", "unleash", "powerful", "seamless", "robust", "delve", "empower", "leverage" as a verb, "in today's world").
- Short sentences. Active voice. Concrete nouns over adjectives.
- Code blocks over prose when a command or path can replace a sentence.
